L'engranatge de la intel·ligència educativa

Com ATNE transforma un text en una adaptació ajustada al perfil d'un alumne, pas a pas. Arquitectura determinista, 96 instruccions, model open-source al free tier, verificació automàtica i transparència total via prompt preview.

01

Els 4 pilars

ATNE no és un assistent conversacional. És un motor de regles deterministes amb un LLM d'execució al final del pipeline. Aquests són els seus quatre pilars tècnics.

Arquitectura determinista

Catàleg de 96 instruccions graduades per MECR i activades per perfil. Sense recuperació dinàmica. El RAG es va desactivar el 2026-04-09 perquè les instruccions cobrien exhaustivament el "què fer".

Model d'execució

Gemma 4 31B o Mistral Small segons env var ATNE_MODEL. Els dos al free tier. Gemma 4 és el recomanat a producció FJE (triangulació empírica).

Verify + Retry

Cada adaptació passa per verificació rúbrica Python. Llindar ≥4.0/5. Si no passa, es regenera. Màxim 2 intents automàtics. Millora la consistència sense intervenció humana.

Prompt Preview

L'endpoint /api/prompt-preview munta el prompt complet sense cridar l'LLM. Cost zero. Per transparència i inspecció del sistema.

02

Flux del pipeline

El pipeline complet en 6 passos, des que el docent enganxa un text fins que rep l'adaptació verificada.

01

El docent configura

Enganxa o puja el text. Defineix el context (etapa, curs, àmbit, matèria, gènere discursiu). Defineix el perfil de l'alumne — via diagnòstic (condicions + situacions + sub-variables) o via observable (5 conductes).

02

El sistema calcula automàticament

DUA (Accés / Core / Enriquiment) auto-computat per severitat de perfil. MECR de sortida calculat des de l'etapa + barreres lingüístiques. Complements activats per regles de negoci.

03

Filtre d'instruccions

De les 96 instruccions del catàleg, se'n seleccionen ~40-60 segons perfil + MECR + sub-variables. Supressió intel·ligent de redundants. Intensificació per gravetat. Agrupació en 7-8 macrodirectives (LÈXIC, SINTAXI, ESTRUCTURA, COGNITIU, QUALITAT, MULTIMODAL, AVALUACIÓ, PERSONALITZACIÓ).

04

Prompt assemblat — 4 capes

Capa 1 Identitat (fixa: "Ets ATNE de Jesuïtes Educació"). Capa 2 Macrodirectives (7-8 blocs temàtics sense IDs visibles). Capa 3 Context (educatiu + persona-audience + blocs del corpus FJE). Capa 4 Format de sortida (## Text adaptat + seccions opcionals).

05

LLM genera

Gemma 4 31B o Mistral Small rep el system prompt + text original. Temperatura 0.4, max output 16.384 tokens. Streaming SSE al frontend. L'LLM no veu els IDs d'instruccions — només macrodirectives temàtiques.

06

Verify + Retry i entrega

Rúbrica Python verifica llargada de frases per MECR, vocabulari prohibit, mètriques. Score <4.0/5 dispara retry automàtic (max 2 intents). El docent rep el text a la columna dreta: editable, amb feedback i export PDF/DOCX/TXT.

03

Memòria triàdica

Tres capes de context que el sistema envia al LLM per entendre per a qui escriu. Avui el context es passa per petició, no es persisteix entre sessions — aquesta memòria és aspiracional, serà completa a la v2.

Alumne

Origen, L1, MECR, alfabetització llatina, condicions i situacions actives, sub-variables fines (grau, severitat, presentació).

Estat: per petició. Pendent: perfil persistent per alumne.

Classe

Configuració multinivell (distribució d'alumnes per nivell DUA), estil de suport preferit del docent, recursos disponibles a l'aula.

Estat: no implementat. Pendent: configuració per grup classe.

Matèria

Assignatura, estil de redacció esperat, termes intocables (nomenclatura binomial a biologia, topònims a geografia...), gèneres discursius propis.

Estat: camp present, no activat per instruccions. Parking lot.

06

Prompt preview (cost zero)

Eina de transparència radical: dona-li una configuració i el sistema munta el prompt complet sense cridar l'LLM. Útil per auditoria, validació docent i debugging.

Capa 1 · Identitat fixa

"Ets ATNE de Jesuïtes Educació. Restriccions: NO inventis contingut. NO meta-text. Preserva termes tècnics. Adapta, no crees."

Capa 2 · Macrodirectives variable — 7-8 blocs temàtics

LÈXIC (vocabulari, termes, referents) · SINTAXI (longitud frase, ordre SVO, connectors) · ESTRUCTURA (paràgrafs, títols, fragmentació) · COGNITIU (chunking, càrrega, bastides) · QUALITAT (no inventar, preservar rigor) · MULTIMODAL (pictogrames, esquemes) · AVALUACIÓ (preguntes, comprensió) · PERSONALITZACIÓ (ajustos perfil)

Capa 3 · Context variable — educatiu + persona + blocs corpus

Context educatiu (etapa, curs, matèria, gènere discursiu) · Persona-audience (narrativa natural de l'alumne: origen, L1, CALP, MECR, necessitats) · Blocs del corpus FJE (DUA block + gènere block + blocs de creuament) · Complements activats (glossari, esquema, preguntes...)

Capa 4 · Format de sortida fixa

## Text adaptat
## Argumentació pedagògica
(## Glossari · ## Esquema visual · ## Mapa conceptual · ## Preguntes de comprensió · ## Bastides · ## Activitats d'aprofundiment)

Generador de prompt preview

A producció, aquesta secció té selectors (perfil, etapa, MECR, DUA, complements) que criden POST /api/prompt-preview i retornen el prompt complet assemblat — sense cridar l'LLM, cost zero. Inclou mètriques (nombre de paraules, tokens estimats, instruccions activades, capes).

07

RAG + Knowledge Graph

Nota: la infraestructura RAG+KG està desactivada des del 2026-04-09. Es manté retenguda per si cal reactivar-la, però el pipeline actual NO fa cerca vectorial. Els blocs del corpus FJE s'injecten directament des dels fitxers Markdown via corpus_reader, no via RAG dinàmic.

Cerca vectorial

1.443 chunks · embeddings 768d · Supabase pgvector · funció match_rag_fje.

Infraestructura a Supabase, no invocada pel pipeline actual.

Knowledge Graph

952 nodes + 2.294 arestes · taules kg_nodes/kg_edges · expansió 1-2 salts via kg_expand.

Infraestructura retenguda, no invocada pel pipeline actual.

08

Com ha evolucionat l'arquitectura

ATNE no ha arribat a l'arquitectura actual de cop. Cada decisió ha estat el resultat d'un experiment concret. Aquestes són les 8 fites més rellevants del projecte, en ordre cronològic.

2026-03-27

Arquitectura inicial — catàleg jeràrquic de 96

El primer catàleg tenia 30 instruccions duplicades i amb conflictes. Una auditoria exhaustiva va produir l'arquitectura actual: identitat fixa + instruccions universals + condicionals per perfil + context. El nucli de 25 instruccions cobria el 80% de l'impacte.

2026-03-28

RAG vs hardcoded — primera aposta per RAG

Un batch de 160 casos va comparar RAG (lectura dinàmica del corpus) vs prompt hardcoded. RAG va guanyar en riquesa: 3× més paraules i 3× més negretes. Es va adoptar com a via principal amb hardcoded com a fallback.

2026-03-30

Auditoria variables — només el 42% connectaven

De 36 sub-variables del formulari, només 15 arribaven al prompt. Es van afegir instruccions noves i gradacions per MECR per connectar-ne més. Actualment el 56% (20/36) estan connectades.

2026-04-04

Gemma 4 free tier — motor sense factura

Es va confirmar Gemma 4 31B via API gratuïta de Google: 30 RPM, 14.400 RPD, 15.000 TPM sense billing. Amb 7 claus rotades es cobreix el consum d'unes 1.000 docents. Canvia radicalment les matemàtiques d'escalat.

2026-04-08

Girada determinista — el RAG es desactiva

Amb el catàleg complet de 96 instruccions, el prompt determinista supera el RAG en qualitat per +0.26 punts. El RAG recuperava referències teòriques que l'LLM ja tenia integrades. La infraestructura RAG es manté retenguda però no s'invoca.

2026-04-11

Terminologia del perfil — condicions + situacions

De "característiques" a condicions (permanents: TEA, TDAH, dislèxia...) + situacions contextuals (transitòries: nouvingut, vulnerabilitat, trastorn emocional). El perfil d'un alumne és la combinació única de diverses.

2026-04-11

Via observable — adaptar sense diagnòstic

Via d'entrada alternativa per a docents sense diagnòstic clínic (60-80% dels casos reals). El docent marca 5 conductes observables i el sistema dedueix perfils aproximats. Les conductes no estigmatitzen — només determinen ajuts al text.

2026-04-12

Triangulació Gemma 4 — 3 experiments independents

Gemma 4 31B guanya en adaptacions (4.75/5), complements (4.58/5) i preguntes de comprensió (4.40/5). Total: 1.884 avaluacions en tres dominis independents. Recomanat com a model per defecte a la migració FJE. Detalls al bloc Avaluació i decisions.
